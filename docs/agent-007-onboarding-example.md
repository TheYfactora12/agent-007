# Agent-007 Onboarding Example

## Bottom Line

This example shows how to use Agent-007 without reading the whole repo first.

Start with one idea. Turn it into one mission packet. Route the risk. Produce one useful artifact. Capture the lesson.

## Example Scenario

Kevin says:

```text
I want to turn my AI security framework into a teachable product for credit union leaders.
```

## Step 1 — Cut Through the Noise

Agent-007 asks:

```text
What matters most right now?
```

Answer:

```text
Define the first target user and create a mission packet. Do not build the full course yet.
```

## Step 2 — Name the Target User

Primary user:

```text
Credit union executives or security leaders who need a practical AI security governance framework.
```

Secondary user:

```text
Kevin, as the instructor/consultant building the product.
```

## Step 3 — Risk Tier

Likely risk:

```text
Medium
```

Reason:

```text
This is product/framework planning. It may later affect public claims and consulting positioning, but no production system, regulated data, or spend is involved yet.
```

## Step 4 — Governance Route

```text
Proceed with Kevin visibility.
```

If public claims, pricing, or launch decisions enter scope, escalate.

## Step 5 — Mission Packet

```md
## Mission

Create the first teachable 4 Walls AI Security Framework module.

## Target User

Credit union executives and security leaders.

## Why It Matters

AI security is confusing. The product should cut through noise and give leaders a practical control-room model.

## Risk Tier

Medium.

## Done Means

A first module outline exists with target user, learning objectives, controls, examples, and risks.

## Stop Conditions

Pause if:
- public claims become overstated
- regulatory claims lack evidence
- the framework becomes too complex to teach
- target user becomes vague

## Evidence Required

- module outline
- plain-language explanation
- risk review
- teaching note
- next action
```

## Step 6 — Produce the Smallest Useful Artifact

Do not build the whole course.

Produce:

```text
One module outline.
```

## Step 7 — Capture QA / Evidence

Check:

- Is the target user clear?
- Is the lesson teachable?
- Are claims defensible?
- Does it help a real person act?
- Does it avoid hype?

## Step 8 — Capture Learning

Learning artifact:

```text
The first teachable product should not start as a full course. It should start as one clear module that proves the framework can be explained simply.
```

## Step 9 — Next Action

```text
Draft Module 1: AI Security Control Room Overview.
```

## Key Takeaway

Agent-007 is working when it turns a broad idea into a specific, risk-routed, teachable next action.

If it produces more confusion, it failed the simplicity rule.
